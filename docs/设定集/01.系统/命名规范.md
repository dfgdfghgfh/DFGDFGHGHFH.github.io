# 命名规范

> 一致性是长期可维护性的前提。本文件定义了全套设定集中所有文件的命名、组织和格式约定。

---

## 一、目录命名

### 顶层目录

格式：`NN.名称/`

- `NN` 为两位数字序号（01~99），决定排序
- 序号之间建议间隔 10 以上，为未来插入留空间
- `名称` 使用中文，尽量简短（2~6 字）

当前顶层目录：

| 序号 | 目录 | 内容 |
|:----:|:-----|:-----|
| 01 | 系统 | 地基文档：通用规则、术语、规范 |
| 02 | 修仙体系扩展 | 修仙 lore 深度展开 |
| 03 | 全新异能体系 | 契灵术完整设定 |
| 04 | 跨体系联动 | 两体系交互点展开 |
| 05 | 叙事与设定 | 世界观背景：地理、历史、组织 |
| 06 | 工具与模板 | 写作工具、检查清单 |
| 99 | 归档 | 废弃或历史版本 |

### 子目录

子目录可不用序号，直接用 `名称/`。如果同一层有多个子目录需排序，建议同一套规则：

- 用 `01_名称/`、`02_名称/` 格式（下划线而非点号，避免和顶层混淆）
- 或直接用 `名称/` 不编号，依赖文件层级的排序

## 二、文件命名

### 格式

`NN_标题.md`

- `NN` 为两位数字序号（01~99），决定文件在本目录内的显示顺序
- 序号与标题之间用下划线 `_` 连接
- `标题` 使用中文，简短明确
- 不要在同一文件名中混用多种命名风格

### 序号分配建议

| 序号 | 用途 |
|:----:|:-----|
| 01~10 | 核心定义文件 |
| 11~30 | 主要展开文件 |
| 31~50 | 次要展开/参考文件 |
| 51~99 | 预留 |

### 索引文件

每个目录应包含一个索引文件（`NN_概览.md` 或 `NN_索引.md`），列出该目录下所有文件及其简要说明。

## 三、Frontmatter 规范

### 必填字段

每个 `.md` 文件都必须包含有效的 YAML frontmatter。示例如下：

```yaml
---
type: 设定|系统        # 文档类型
体系: 修仙|契灵|通用|跨体系  # 所属体系
版本: 1.0              # 语义化版本号
date: 2026-07-19       # 最后更新日期
---
```

### 类型枚举

| 取值 | 说明 |
|:----|:-----|
| 设定 | 世界观设定类文档（如底层规则、联动总纲） |
| 系统 | 方法论/工具类文档（如术语表、命名规范） |
| 叙事 | 故事/场景类文档 |
| 模板 | 可复用的格式模板 |

### 可选字段

```yaml
status: 草稿|初稿|定稿   # 完成度状态
author: 镜               # 默认作者
tags: [tag1, tag2]      # 标签（尽量少用，优先依赖目录结构）
cssclass: class-name    # Obsidian 样式类（用于特殊渲染）
```

### 字段填写规则

- 版本号格式：`major.minor`（如 `1.0`、`2.3`）
- 日期格式：`YYYY-MM-DD`
- 体系字段：仅选一个值，多体系文档使用 `跨体系`

## 四、Wiki-link 语法

### 基本规则

- **链接目标**：优先使用绝对路径（从 vault 根目录开始），路径以 `文件夹/文件名` 格式书写。
- **省略 `.md` 后缀**：Obsidian wiki-link 默认省略文件扩展名。
- **显示文本**：使用 `显示文本` 语法控制显示的文本内容。

### 约定

| 场景 | 语法 | 说明 |
|:----|:-----|:-----|
| 链接本体系文件 | `[术语对照表](/01.系统/术语对照表.md\)` | 推荐省略 `.md`，但实际使用中带不带均可 |
| 链接外部体系文件 | `[零之书](/修仙养成/数据/零之书.md\)` | 跨目录链接写完整路径 |
| 链接到锚点 | `[道](/01.系统/术语对照表.md#道\)` | 当需跳转到文件中某个具体段 |
| 嵌入 | `![01.系统/术语对照表.md](/01.系统/术语对照表.md)` | 嵌入整个文件（慎用，会影响加载速度） |

### 注意事项

- 链接目标文件时，若目标文件在同一目录下，可用相对路径；推荐**始终使用 vault 根目录下的完整路径**以避免歧义。
- 避免创建指向不存在的文件的链接（除非有意识预留）。
- 创建新链接后，如果目标文件尚未创建，可在文件名后标注 `（待创建）` 以标识状态。

## 五、术语格式化规则

### 首次出现

每个术语在文件中第一次出现时，使用 wiki-link 指向 [01.系统/术语对照表.md](/01.系统/术语对照表.md) 中的对应定义：

```markdown
[道](/01.系统/术语对照表.md)是世界运行的底层规律。
```

### 再次出现

同一文件中再次提及同一术语时，用 **粗体** 标明：

```markdown
**道**不可名状，但可通过修行感知。
```

### 术语对照表自身

术语对照表中的术语条目遵循固定格式，见 [01.系统/术语对照表.md](/01.系统/术语对照表.md)。

### 英文术语

- 首次出现时用括号标注英文，如：修仙（Cultivation）
- 之后可用中文或英文，保持全文一致
- 专有名词首字母大写，普通名词全部小写

## 六、版本历史格式

### 版本号规则

- 初始版本为 `1.0`
- 版本号格式：`major.minor`
- major 递增：重大内容变更（新增/删除体系级内容、重构目录结构）
- minor 递增：小幅修订（增补条目、修正错误、优化表述）

### 变更记录

每份文件的版本历史记录在 [01.系统/版本历史.md](/01.系统/版本历史.md)。单次变更的格式：

```
v{major}.{minor} ({YYYY-MM-DD})：{变更描述}
```

示例：

```
v1.1 (2026-08-01)：增补契灵体系 5 条术语。修订跨体系联动第三条规则表述。
```

## 七、文件头注释

每个文件开头的 frontmatter 之后、正文之前，可添加一条引语（blockquote）说明本文件的性质：

```markdown
> 一句话说明本文件的用途。语气保持客观、冷静，符合镜的叙事风格。
```

示例：

```
> 一切设定的起点。若非本文件所载，则非此宇宙之真相。
```

非必要，但推荐。

---

## 八、章节编号

- 使用中文数字：一、二、三……（或 一、1.、1.1 的层级）
- 不使用纯数字如 1.1.1——因为 Obsidian 标题层级（## / ###）本身已提供结构
- 用 `##` 和 `###` 表示章节，不使用 `#` 以外的更高层级
- 章节号与标题之间空一格：`## 一、项目定位`

---

## 九、元规范

本规范自身**必须被遵守**。如果发现本规范未覆盖的情况，先讨论补充规范，再按新规范执行。不要创建"例外"——特例会积累成混乱。
